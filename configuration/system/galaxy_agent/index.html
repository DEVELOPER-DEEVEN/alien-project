cluster network Agent Configuration

agent.yaml configures the network Agent - the AI agent responsible for creating networks (task decomposition) and editing them based on execution results.

Overview

The agent.yaml configuration file provides LLM and API settings for the network Agent. This agent is responsible for:

  • network Creation: Breaking down user requests into device-specific tasks
  • network Editing: Adjusting task plans based on execution results
  • Device Selection: Choosing appropriate devices for each sub-task
  • Task Orchestration: Coordinating multi-device workflows

Configuration Separation:

  • agent.yaml - LLM configuration for network agent (this document)
  • network.yaml - Runtime settings for orchestrator (cluster network Configuration)
  • devices.yaml - Device definitions (cluster Devices Configuration)

Agent Role in System:

graph TB A[User Request] -->|Natural Language| B[network Agent] B -->|Uses LLM Config| C[agent.yaml] B -->|Creates/Edits| D[network Plan] D -->|Tasks| E[Device Agent 1] D -->|Tasks| F[Device Agent 2] D -->|Tasks| G[Device Agent N] style B fill:#e1f5ff style C fill:#ffe1e1 style D fill:#fff4e1

File Location

Standard Location:

Alien-Unis/
├── config/
│   └── cluster/
│       ├── agent.yaml              # ← network agent config (copy from template)
│       ├── agent.yaml.template     # ← Template for initial setup
│       ├── network.yaml      # ← Runtime settings
│       └── devices.yaml            # ← Device definitions

Setup Required

  1. Copy agent.yaml.template to agent.yaml
  2. Fill in your API credentials (API_KEY, AAD_TENANT_ID, etc.)
  3. Never commit agent.yaml with real credentials to version control

Loading in Code:

from config.config_loader import get_cluster_config

# Load cluster configuration (includes agent settings)
config = get_cluster_config()

# Access network agent settings
agent_config = config.network_agent
reasoning_model = agent_config.reasoning_model
api_type = agent_config.api_type
api_model = agent_config.api_model

Configuration Schema

Complete Schema

# cluster network Agent Configuration

network_AGENT:
  # Reasoning
  REASONING_MODEL: bool          # Enable reasoning/chain-of-thought

  # API Connection
  API_TYPE: string               # API provider type
  API_BASE: string               # API base URL
  API_KEY: string                # API authentication key
  API_VERSION: string            # API version
  API_MODEL: string              # Model name/deployment

  # Azure AD Authentication (for azure_ad API_TYPE)
  AAD_TENANT_ID: string          # Azure AD tenant ID
  AAD_API_SCOPE: string          # API scope name
  AAD_API_SCOPE_BASE: string     # API scope base GUID

  # Prompt Configuration
  network_CREATION_PROMPT: string         # Path to creation prompt template
  network_EDITING_PROMPT: string          # Path to editing prompt template
  network_CREATION_EXAMPLE_PROMPT: string # Path to creation examples
  network_EDITING_EXAMPLE_PROMPT: string  # Path to editing examples

Configuration Fields

Reasoning Capabilities

Field Type Required Default Description
REASONING_MODEL bool No False Enable chain-of-thought reasoning for complex planning

Example:

network_AGENT:
  REASONING_MODEL: False  # Standard LLM response (faster)

Reasoning Model

Set REASONING_MODEL: True for: - Complex multi-device workflows - Tasks requiring step-by-step planning - Debugging network failures

Trade-off: Slower response time, higher token cost

API Connection Settings

Field Type Required Default Description
API_TYPE string Yes - API provider: "openai", "azure", "azure_ad", "aoai"
API_BASE string Yes* - API base URL (required for Azure)
API_KEY string Yes* - API authentication key (required for non-AAD auth)
API_VERSION string Yes* - API version (required for Azure)
API_MODEL string Yes - Model name or deployment name

Supported API Types:

API_TYPE Provider Authentication Example API_BASE
openai OpenAI API Key Not required (uses default)
azure Azure OpenAI API Key https://your-resource.openai.azure.com/
azure_ad Azure OpenAI Azure AD (AAD) https://your-resource.azure-api.net/
aoai Azure OpenAI (alias) API Key https://your-resource.openai.azure.com/

Example 1: OpenAI Configuration

network_AGENT:
  API_TYPE: "openai"
  API_KEY: "sk-proj-..."           # Your OpenAI API key
  API_MODEL: "gpt-4o"              # OpenAI model name
  API_VERSION: "2024-02-01"        # Optional for OpenAI

Example 2: Azure OpenAI (API Key Auth)

network_AGENT:
  API_TYPE: "azure"
  API_BASE: "https://my-resource.openai.azure.com/"
  API_KEY: "abc123..."             # Azure OpenAI API key
  API_VERSION: "2025-02-01-preview"
  API_MODEL: "gpt-4o-deployment"   # Your deployment name

Example 3: Azure OpenAI (Azure AD Auth)

network_AGENT:
  API_TYPE: "azure_ad"
  API_BASE: "https://cloudgpt-openai.azure-api.net/"
  API_VERSION: "2025-02-01-preview"
  API_MODEL: "gpt-5-chat-20251003"

  # Azure AD Configuration
  AAD_TENANT_ID: "72f988bf-86f1-41af-91ab-2d7cd011db47"
  AAD_API_SCOPE: "openai"
  AAD_API_SCOPE_BASE: "feb7b661-cac7-44a8-8dc1-163b63c23df2"

Azure AD Authentication

When using API_TYPE: "azure_ad": - No API_KEY needed (uses Azure AD token) - Requires AAD_TENANT_ID, AAD_API_SCOPE, AAD_API_SCOPE_BASE - User must be authenticated with az login or have proper AAD credentials

Azure AD Fields (azure_ad API_TYPE only)

Field Type Required Description
AAD_TENANT_ID string Yes* Azure AD tenant GUID
AAD_API_SCOPE string Yes* API scope identifier (e.g., "openai")
AAD_API_SCOPE_BASE string Yes* API scope base GUID

*Required only when API_TYPE: "azure_ad"

Prompt Configuration Paths

Field Type Required Default Description
network_CREATION_PROMPT string Yes - Path to network creation prompt template
network_EDITING_PROMPT string Yes - Path to network editing prompt template
network_CREATION_EXAMPLE_PROMPT string Yes - Path to creation examples (few-shot learning)
network_EDITING_EXAMPLE_PROMPT string Yes - Path to editing examples (few-shot learning)

Default Prompt Paths:

network_AGENT:
  network_CREATION_PROMPT: "cluster/prompts/network/share/network_creation.yaml"
  network_EDITING_PROMPT: "cluster/prompts/network/share/network_editing.yaml"
  network_CREATION_EXAMPLE_PROMPT: "cluster/prompts/network/examples/network_creation_example.yaml"
  network_EDITING_EXAMPLE_PROMPT: "cluster/prompts/network/examples/network_editing_example.yaml"

Custom Prompts

You can customize prompts for your use case: 

network_CREATION_PROMPT: "custom_prompts/my_network_creation.yaml"

Security Best Practices

Never Commit Credentials

DO NOT commit agent.yaml with real credentials to version control!

✅ Recommended Workflow: 

# 1. Copy template
cp config/cluster/agent.yaml.template config/cluster/agent.yaml

# 2. Edit agent.yaml with your credentials
# (This file is .gitignored)

# 3. Commit only the template
git add config/cluster/agent.yaml.template
git commit -m "Update agent template"

Use Environment Variables for Sensitive Data:

# In agent.yaml
network_AGENT:
  API_KEY: ${cluster_API_KEY}  # Read from environment variable

# In your shell
export cluster_API_KEY="sk-proj-..."

Integration with Other Configurations

The agent configuration works together with other cluster configs:

agent.yaml (LLM config) + network.yaml (runtime) + devices.yaml (devices) → Complete cluster System

Complete Initialization Example

from config.config_loader import get_cluster_config
from cluster.agents.network_agent import networkAgent
from cluster.client.device_manager import networkDeviceManager
import yaml

# 1. Load all cluster configurations
cluster_config = get_cluster_config()

# 2. Initialize network Agent with LLM config
agent = networkAgent(
    reasoning_model=cluster_config.network_agent.reasoning_model,
    api_type=cluster_config.network_agent.api_type,
    api_base=cluster_config.network_agent.api_base,
    api_key=cluster_config.network_agent.api_key,
    api_version=cluster_config.network_agent.api_version,
    api_model=cluster_config.network_agent.api_model
)

# 3. Load network runtime settings
with open("config/cluster/network.yaml", "r") as f:
    network_config = yaml.safe_load(f)

# 4. Initialize Device Manager with runtime settings
device_manager = networkDeviceManager(
    task_name=network_config["network_ID"],
    heartbeat_interval=network_config["HEARTBEAT_INTERVAL"],
    reconnect_delay=network_config["RECONNECT_DELAY"]
)

# 5. Load and register devices
device_config_path = network_config["DEVICE_INFO"]
with open(device_config_path, "r") as f:
    devices_config = yaml.safe_load(f)

for device in devices_config["devices"]:
    await device_manager.register_device(**device)

print("✅ cluster network System Initialized")
print(f"   Agent Model: {cluster_config.network_agent.api_model}")
print(f"   network ID: {network_config['network_ID']}")
print(f"   Devices: {len(devices_config['devices'])}")

Best Practices

Configuration Best Practices:

  1. Use Templates for Team Collaboration 

    # Share template, not credentials
config/cluster/agent.yaml.template  # ✅ Commit this
config/cluster/agent.yaml           # ❌ Never commit this

  2. Test with OpenAI, Deploy with Azure 

    # Development: OpenAI (fast iteration)
API_TYPE: "openai"

# Production: Azure (enterprise features)
API_TYPE: "azure_ad"

  3. Use Reasoning Mode Selectively 

    # For complex workflows
REASONING_MODEL: True

# For simple tasks
REASONING_MODEL: False  # Faster
